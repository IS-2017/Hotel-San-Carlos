-SE DEBE CREAR UN DSN LLAMADO "prueba2"

-SE RECOMIENDA CREAR UN NUEVO USUARIO QUE POSEA TODOS LOS PERMISOS PARA NO TENER NINGUN INCONVENIENTE A LA HORA DE PROBAR
LA APLICACION, PARA ELLO SE DEBERÁ CREAR UN USUARIO NUEVO:

 create user administrador identified by 'aABvAGwAYQA=' 
LA CONTRASEÑA DEBE INGRESARSE ENCRIPTADA Y AL MOMENTO DE INGRESAR LA CONTRASEÑA EN EL LOGIN SE DEBERÁ ESCRIBIR "hola".

EL USUARIO DEBE TAMBIEN INGRESARSE EN LA TABLA USUARIO CON LA CONTRASEÑA ENCRIPTADA:
 insert into usuario (usuario, contrasenia) values ('administrador','aABvAGwAYQA=');

Y POR ULTIMO SE LE DEBERÁ DAR TODOS LOS PERMISOS:
GRANT ALL PRIVILEGES ON sancarlos2.* TO 'administrador'@'%' WITH GRANT OPTION;
GRANT EXECUTE, PROCESS, SELECT, SHOW DATABASES, SHOW VIEW, ALTER, ALTER ROUTINE,
    CREATE, CREATE ROUTINE, CREATE TEMPORARY TABLES, CREATE VIEW, DELETE, DROP,
    EVENT, INDEX, INSERT, REFERENCES, TRIGGER, UPDATE, CREATE USER, FILE,
    LOCK TABLES, RELOAD, REPLICATION CLIENT, REPLICATION SLAVE, SHUTDOWN,
    SUPER
        ON *.* TO administrador@'%'
    WITH GRANT OPTION;
flush privileges

*SI SE UTILIZA WAMP O XAMP DEBERAN CORRERSE ESTOS MISMOS QUERYS PERO CON 'localhost' en lugar de '%' DESPUES DE LA ARROBA
 , CON ESTO TENEMOS LISTO NUESTRO USUARIO ADMINISTRADOR CON ACCESO A TODO.

-EL USUARIO "DUMPY" ES UN USUARIO NECESARIO PARA EL CORRECTO FUNCIONAMIENTO DE LA SEGURIDAD, POR ALGUNA RAZÓN CUANDO
SE EXPORTA LA BASE DE DATOS JUNTO CON ESTE USUARIO EN OCASIONES NO SUELE FUNCIONAR BIEN POR LO QUE SE HA CONFIGURADO 
LA DLL DE SEGURIDAD QUE LLEVA ESTE PROYECTO PARA FUNCIONAR CON EL USUARIO "seg" EN LUGAR DEL DUMPY, EL CUAL NO VIENE
EN EL EXPORT DE LA BASE DE DATOS CON EL FIN DE QUE NO SE "ARRUINE" POR LO QUE DEBERÁ CREARSE Y OTORGARSELE TODOS LOS
 PERMISOS:

create user seg identified by 'seg'  
GRANT ALL PRIVILEGES ON sancarlos2.* TO 'seg'@'%' WITH GRANT OPTION;
GRANT EXECUTE, PROCESS, SELECT, SHOW DATABASES, SHOW VIEW, ALTER, ALTER ROUTINE,
    CREATE, CREATE ROUTINE, CREATE TEMPORARY TABLES, CREATE VIEW, DELETE, DROP,
    EVENT, INDEX, INSERT, REFERENCES, TRIGGER, UPDATE, CREATE USER, FILE,
    LOCK TABLES, RELOAD, REPLICATION CLIENT, REPLICATION SLAVE, SHUTDOWN,
    SUPER
        ON *.* TO seg@'%'
    WITH GRANT OPTION;
flush privileges

*DE IGUAL FORMA SI SE USA XAMP O WAMP DEBERÁ DARSE LOS PERMISOS A NIVEL DE LOCALHOST Y %
*PARA EL USUARIO seg  NO ES NECESARIO QUE LA CONTRASEÑA ESTE ENCRIPTADA
